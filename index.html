MLOps Central CI/CD — Multi-Cloud Infrastructure
CI/CD · MLOps · DevSecOps

Eine produktionsreife MLOps-Plattform, die Modelle über AWS, Azure und GCP ausliefert — ohne ein einziges gespeichertes Credential, ohne manuelle Key-Rotation, ohne Kompromisse bei der Sicherheit auf irgendeiner Ebene.

  • Zentrales GitLab-Repo-Pattern — 5 autonome Repos (model-build, model-deploy, infrastructure, inferenz, model-registry) konvergieren in einem einzigen Orchestrierungspunkt mit vollständiger Isolation zwischen den Domains
  • Keyless OIDC/JWT-Authentifizierung gegen AWS STS, Azure Entra ID und GCP STS — ein JWT pro Job, zur Laufzeit generiert, nach ~1h automatisch abgelaufen, nichts gespeichert
  • EC2 Self-hosted Runner — bewusste Wahl (Azure VM oder GCP Compute Engine wären äquivalent) mit VPN/WireGuard-Tunnel zu privaten Cloud-Ressourcen und IAM Least-Privilege pro Job
  • Defense-in-Depth über 5 unabhängige Sicherheitsebenen: VPN/WireGuard · TLS 1.2+ · OIDC/JWT · IAM Least-Privilege · Secrets Manager (HashiCorp Vault / AWS SSM / Azure Key Vault / GCP Secret Manager)
  • Obligatorische Pipeline-Gates: check → analyse → apply → deploy — keine Stage kann übersprungen werden, kein Deployment ohne vollständigen Audit-Trail
  • 0 credentials gespeichert — 100% keyless Authentifizierung über 3 Clouds gleichzeitig
